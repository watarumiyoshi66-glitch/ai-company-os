### SH-003: 外資IT・AI領域のビジネスパーソン

**現状**
- 認知: X経由で約30%、Note経由で約50%、検索流入で約20%（推定）
- 購買意思: 認知層だが、まだ自社サービス・コンサルの購買層ではない

**期待事項**
- 「自分の会社でも応用できそう」と感じるケーススタディが欲しい
- 専門家ぶらず、凡人営業マンが試行錯誤している内容に信頼感
- すぐ実装できるツール・フレームワークの情報

**影響度が中程度な理由**
- 短期的には直接収益にならない（Month 6以降の営業パイプライン）
- ただし良質な紹介者・アライアンスパートナーになる可能性が高い
- 法人顧客への信頼構築の基盤になる

**エンゲージメント戦略の実行方法**
1. **メッセージフレーム**: 「凡人営業マンのAI導入実験室」
   - 「AI専門家ではない自分たちでも、こんなふうに試している」という視点を一貫維持
   - HBR・McKinsey 等の難しい論文を「営業目線で噛み砕く」コンテンツを定期配信
2. **具体化施策**:
   - 月1回、「某外資SaaS の300名規模製造業向け案件」のような業種・規模別ケーススタディ
   - 「このツール、営業でも使える」という実装例を紹介
3. **タッチポイント**:
   - X: 週2回、業界ニュース + 自社実験結果をシェア
   - Note: 月1本の深掘り記事で、導入プロセス全体を公開

**責任者**: Research Analyst（トレンド調査）、Content Writer（ケーススタディ執筆）

- ターゲット規模: 従業員100名〜1000名の中堅・大手企業IT部門、営業部門

**期待事項**

- どのくらいコストがかかり、どのくらい時間がかかるのか知りたい（実装コスト）
- 失敗リスク・競合他社の事例を知りたい（リスク評価）

**影響度が高い理由**
- Month 6以降の顧問・コンサル収益の源泉（最大の長期収益）
- 複数顧客獲得で事業の再現性が証明される

**現在の関与度が認知前段階な理由**
- Xでオーナーのアカウントはまだ法人向け認知が低い
- Note記事もB2C向けが主で、B2B向けユースケースが不足している

**エンゲージメント戦略の実行方法**

*フェーズ1 (Month 1-2): 認知構築*
- Xで「AI導入の全体像」を3部シリーズで発信
- 自社で導入した実装例をケーススタディとして段階的に公開

*フェーズ2 (Month 3-4): 検討層への展開*
- Note で「AI導入企業向け実装ガイド」を有料記事化
- 「お困りですか？」という問題提示型の投稿でDM誘導

*フェーズ3 (Month 5-6): 案件化*
- Xプロフィール欄に「AI導入コンサル」を明記
- 簡易LP（GitHubやnote.com のランディングページ機能）で問い合わせ窓口を設置
- 月2回の「無料相談受付中」投稿で接触機会を生成

*継続的なタッチ*
- 月1回、「企業でのAI導入事例」（実名は避け、業種・規模のみ）を記事化
- DM での問い合わせに1営業日以内で初期対応

**責任者**: オーナー（営業判断）、Content Writer（ケーススタディ作成）、Sales Strategist（提案資料準備）

**現状**

- Daily Plan で毎日タスク指示
- Repository 内のプロンプト・仕様書で行動基準を管理

**期待事項**
- 明確な業務指示（曖昧さのない Daily Plan）
- 判断基準が明示されていること（仕様書・ルール）
- 実行タスクと判断タスクの分離

**影響度が高い理由**
- オーナーの時間を解放し、判断に集中させるための鍵
- AI社員の能力 = 副業事業の実行スピード・品質を決める

**エンゲージメント戦略の実行方法**
1. **Daily Plan**: 朝、Daily Plan をRepository に保存し、意思決定を明記
   ```
   ## 今日の判断事項
   - [ ] X投稿テーマ「〇〇」で良いか？
   - [ ] Note記事の対象読者は「B2B」か「B2C」か？
   ```
2. **Daily Review**: 夜、実行結果をRepository に保存し、明日の優先度を調整
3. **月1回の能力評価**: 各AI社員が「今月のコンテンツ品質・スピード」を自己評価し、改善案を提出
4. **仕様書の更新**: 運用を通じて学んだベストプラクティスを `/agents/specs/` に反映

**責任者**: オーナー（指示・判断）

- 副業: AI Company OS（週10〜15時間を目標）
- スキル: 営業ノウハウ、AI活用実験、戦略思考

**意思決定領域**
- X投稿 / Note記事のテーマ選定
- 有料化・値段設定
- 法人営業の優先度・営業メッセージ
- AI社員の能力強化・スコープ変更
- 大きな方針変更

**戦略上の関心事**
- X: フォロワー数 1000→5000→10000 への段階的成長
- Note: 購読者数と有料化比率のバランス
- 法人営業: パイプライン化までのリードタイム

1. **Daily Plan（毎朝）**: AI社員が「今日のやること + 判断待ち」を明記

   - 判断内容は Daily Plan に記録され、AI社員の判断基準になる
2. **Daily Review（毎夜）**: 実行結果を確認し、「明日どうするか」を決定
   - ただし Review 作成は AI社員が行い、オーナーは5分で確認・承認
3. **Weekly Strategy Review（毎日曜）**: 週次で以下を確認
   - 成長指標（フォロワー数、PV、コンバージョン）
   - 来週のテーマ・優先度
   - ボトルネック・改善施策
4. **月1回のマイルストーン確認**: 
   - Month 1-2: Content Velocity（投稿数・記事数）の確保
   - Month 3-4: エンゲージメント率の改善
   - Month 5-6: 初回法人営業機会の創出

**責任者**: オーナー（最高意思決定）

## 3. フィードバックループ設計

**AI Company OS のコンテンツは、読者の反応に基づいて進化する。**

```
┌─────────────────────────────────────────────────────────┐
│            X・Note投稿 & エンゲージメント               │
│  （毎日の投稿、コメント、いいね、RT、保存数）          │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ↓
┌─────────────────────────────────────────────────────────┐
│         Growth Analyst が週次で集計・分析              │
│  ├─ X投稿: いいね/RT/返信数 Top 5               │
│  ├─ Note: PV/開封率/有料購読数                    │
│  ├─ 反響テーマの抽出（「AI」「営業」「SaaS」等）  │
│  └─ 保存先: /reports/growth/YYYY-MM-DD...md      │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ↓
┌─────────────────────────────────────────────────────────┐
│      Weekly Strategy Review でオーナーが確認           │
│  ├─ 高反響テーマは「来週も深掘りするか」判断        │
│  ├─ 低迷テーマは「やめるか工夫するか」判断          │
│  └─ 次週の優先テーマを確定                          │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ↓
┌─────────────────────────────────────────────────────────┐
│    Content Writer が優先テーマで下書き作成             │
│  ├─ Next Week の Daily Plan に組み込む              │
│  ├─ 構成案・リサーチを提出                          │
│  └─ オーナーの承認を待つ                            │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ↓
              （ループ）
```

**記録・実行方法**

| フェーズ | 担当 | 記録先 | 頻度 | 成果物 |
|---|---|---|---|---|
| **反応集計** | Growth Analyst | `/reports/growth/YYYY-MM-DD-growth-report.md` | 週1回（日曜）| いいね数・RT数・コメント数・高反響テーマリスト |
| **戦略レビュー** | オーナー + AI社員 | `/reports/YYYY-MM-DD-weekly-strategy.md` | 週1回（日曜） | 優先テーマ・来週の注力分野・ボトルネック分析 |
| **コンテンツ企画** | Content Writer + Research Analyst | `/workflows/weekly/content-planning.md` | 週1回（月曜） | 来週のX投稿テーマ・Note記事トピック・リサーチ素材 |
| **実行** | Content Writer | `/content/x/drafts/`、`/content/note/drafts/` | 毎日 | 投稿下書き・記事下書き |

**フィードバックの種類と活用**

1. **数値フィードバック**（定量）
   - いいね数・RT数・返信数・PV・購読者数
   - 傾向: 前週比、月間推移を追跡
   - 活用: 「このテーマは前週比150%」→ 来週も同テーマで深掘り

2. **質的フィードバック**（定性）
   - コメント内容・DM の質問・Note のコメント
   - 抽出: 「同じ質問が3件以上あれば、テーマ化候補」
   - 活用: 「営業での AI ツール活用について質問が集中」→ Note でハウツー記事化

3. **接触フィードバック**（行動）
   - スクリーンショット保存（X）、ハイライト登録（Note）
   - 意味: 「後で読み返したい」という強いシグナル
   - 活用: 保存数が多いテーマ = 実装ガイド化の候補

## 5. ステークホルダー管理ルール

### 5.1 コミュニケーション・エスカレーション

**X / Note コメント・DM への対応**

| 優先度 | 種別 | 対応者 | 期限 | 対応内容 |
|---|---|---|---|---|
| **高** | 法人からの問い合わせ・相談DM | オーナー | 1営業日以内 | 初期対応・課題ヒアリング |
| **高** | 購読者からのNote記事へのコメント（建設的） | Content Writer | 24時間以内 | 感謝・補足情報の返信 |
| **中** | X投稿への質問コメント | オーナー | 24時間以内 | 回答・感謝 |
| **中** | Note記事への改善提案 | Content Writer | 48時間以内 | 検討結果の共有（反映 or 理由説明） |
| **低** | スパム・営業DM | AI社員（フィルタリング） | 無対応 | ブロック（必要に応じて） |

**批判・炎上リスク対応**

1. **発見**: Content Writer or AI社員が「批判的なコメント」を発見したら、すぐにオーナーに報告
2. **判断**: オーナーが以下を判断
   - 「正当な指摘か」「建設的な批判か」→ 回答する
   - 「単なる論難・人格攻撃か」 → 無視 or 削除
3. **対応**: 
   - 返信する場合、24時間以内に落ち着いた返信を投稿
   - 削除する場合、理由を簡潔に説明

**個人情報保護**

- [ ] すべてのコンテンツで、実在する顧客・案件・社名は一切記載しない
- [ ] 営業事例は「業種・規模・状況」のみ（例：「某外資SaaS、従業員300名の製造業」）
- [ ] 社内情報・提案書・価格情報は Repository に保存しない
- [ ] 個人が特定できる情報（氏名・連絡先・SNS）は匿名化する
- [ ] 不明な場合は、記載前にオーナーに確認

### 5.2 コンテンツチェック・承認フロー

**公開前のレビュー（すべてのX投稿・Note記事）**

```
Content Writer が下書き作成
        ↓
（内容確認）
        ↓
  個人情報漏洩がないか？
  実例は匿名化されているか？
  不正確な情報がないか？
        ↓
  問題なし → オーナーに提出
  問題あり → Content Writer に修正依頼
        ↓
オーナーが最終確認・承認
  ├─ トーン・内容の確認
  ├─ 戦略的位置づけの確認
  └─ 「これを公開して大丈夫か」の最終判断
        ↓
  承認 → 投稿・公開
  修正 → Content Writer に戻す
```

**特に注意すべきチェック項目**

- [ ] 顧客名・案件名は完全に匿名化されているか
- [ ] 技術情報は正確か（デマや不確実な情報を含んでいないか）
- [ ] リンク先は有効か（リンク切れがないか）
- [ ] X投稿は 140字以内か、Note記事は適切な長さか
- [ ] CTA は適切に配置されているか（過度な営業色がないか）

## 6. ステークホルダー分析マトリックス

```
            関与度
              ↑
              │ 
         高   │  ┌─────────────────────┐
              │  │  SH-006 (オーナー)   │
              │  │  SH-005 (AI社員)    │
              │  │  SH-002 (Note読者)   │
              │  │  SH-004 (法人顧客)   │
              │  └─────────────────────┘
              │
         中   │  ┌─────────────────────┐
              │  │ SH-001 (Xフォロワー) │
              │  │ SH-003 (外資IT層)    │
              │  └─────────────────────┘
              │
         低   │
              │
              └────────────────────────────→ 影響度
                    低      中      高     最高
```

**マトリックス解釈**

- **右上（高影響度 × 高関与度）**: SH-006, SH-005, SH-002, SH-004 → **重点管理対象**
  - 毎日/毎週の定期的なタッチが必須
  - チェックリスト・KPI で進捗追跡
  
- **左上（低影響度 × 高関与度）**: SH-001, SH-003 → **段階的強化対象**
  - 現在は「受動的」だが、エンゲージメント向上で影響度が上昇する可能性
  - 3ヶ月ごとに「影響度の変化」を再評価
