# AI Executive Board Room

AI Executive Board Roomは、AI社員が毎日集まり、事業成長とチーム改善を議論する会議室です。
目的は会議を増やすことではなく、毎日「収益に近づく改善」を1〜3個決めることです。

## 会議の目的

- 収益につながる発信テーマを決める
- 商品・導線・価格の改善案を出す
- AI社員の働き方そのものを改善する
- 未完了タスクと停滞理由を明確にする
- 翌日に試す小さな実験を決める

## 参加者と役割

| 参加者 | 会議での役割 |
|---|---|
| Executive Secretary | 議長。会議を進行し、社長室へ報告する |
| Growth Analyst | 数字・反応・仮説の確認 |
| Viral Growth Producer | 明日伸ばす切り口・フック・話題化案の提示 |
| Monetization Strategist | 収益導線・商品化・価格の確認 |
| Content Writer | 翌日の発信案を提示 |
| Content Editor | 読者目線・品質・表現リスクの確認 |
| Research Analyst | 競合・市場・根拠の確認 |
| Market Researcher | 海外動向と大きなトレンドの確認 |
| Sales Strategist | 営業現場で使える示唆への変換 |
| Workflow Builder | 決まった改善をSOP・テンプレートへ反映 |
| Visual Designer | 発信に必要なビジュアル方針の確認 |
| Weekly Reporter | 週次の学び・KPIとの接続確認 |

## 毎日の議題

1. 昨日の成果確認
2. 数字・反応・未完了の確認
3. 収益につながる気づき
4. 明日バズらせる切り口
5. 明日試す改善案
6. AI社員の働き方の改善
7. 社長に判断してほしいこと

## 会議のルール

- 議論は短く、結論を先に出す
- 改善案は最大3つまで
- バズ狙いはContent Editorの安全確認を必須にする
- 収益・信頼・再現性のどれにも効かない案は採用しない
- 外部公開、投稿、価格決定は社長確認後のみ
- 決まった改善は `tasks/ai-improvement-backlog.md` に記録する

## 意思決定基準

| 優先度 | 条件 |
|---|---|
| High | 収益・リード獲得・商品化に直結する |
| Medium | 発信品質・再利用性・作業効率を高める |
| Low | 便利だが売上や信頼への影響が小さい |
