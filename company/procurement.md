## 1. 調達方針

### 基本原則
- **無料優先**：有料プランは ROI が明確に証明できる場合のみ採用
- **ロックイン回避**：ベンダー依存を最小化し、Markdown + GitHub を Source of Truth に
- **プライバシー厳格**：機密情報・APIキーは Repository に保存しない
- **月次レビュー**：毎月 1 日に費用対効果を確認し、廃止判定を実施

### ツール評価基準（4軸）

| 軸 | 評価項目 |
|---|---|
| **機能** | 事業目標達成に必須かどうか、代替方法の有無 |
| **コスト** | 月額負担、初期投資、中止時の移行コスト |
| **代替可能性** | 同等機能を持つ他ツールの有無、切り替え難易度 |
| **データプライバシー** | 機密情報の暴露リスク、ポリシーの透明性 |

### 新規導入フロー
1. 試用期間を 30 日設定（無料プランで開始）
2. 導入判断書を `/company/cost-management.md` に記録
3. 月額費用が発生する場合、必ず ROI 計算式を記載
4. 3 ヶ月後に効果測定し、継続 / 廃止を判定

### 凡例
- **依存度「高」**：廃止時に事業継続が困難
- **依存度「中」**：代替ツール移行で対応可能（1〜2 週間）
- **依存度「低」**：廃止時の影響最小限

## 3. ベンダー依存リスク評価

### 依存度「高」ツール詳細分析

#### Claude Code（Anthropic）
**現状**
- メイン実行エンジン。日次タスク・コンテンツ作成・データ分析をすべて実行

**代替シナリオ別 移行コスト**

| シナリオ | 移行先 | 所要時間 | 作業量 | 備考 |
|---|---|---|---|---|
| **ChatGPT Pro** に完全移行 | OpenAI | 2 週間 | 中 | API エラーが多い傾向。Claude の方が日本語・複雑タスク向き |
| **Claude API** 自体で運用 | Anthropic SDK | 1 週間 | 低 | コスト削減（月額数千円→数万円で充分）。スクリプト化可能 |
| **ハイブリッド戦略** | ChatGPT + Claude API | 2 週間 | 中 | リスク分散。推奨シナリオ |
| **完全自前化** | Ollama / LLaMA 2 on GPU | 3 週間 | 高 | インフラ構築コスト。中期的に検討 |

**リスク軽減策**
- ✅ Markdown ベース知識管理で、プロンプト・ワークフローは再利用可能
- ✅ CLAUDE.md + tasks/ に運用ルール記録。新 AI へ移行時のプロンプトエンジニアリング工数削減
- ⚠️ 定期的に Claude API 価格を監視し、Anthropic コストシフトに備える

- ⚠️ X Premium 有料化による ROI が低下した場合、即座に廃止判定を実施

## 4. 調達変更管理プロセス

### 4.1 新規ツール追加の承認基準

**以下をすべて満たす場合のみ追加を検討**

```
□ 無料プランで 30 日間試用済み
□ ROI 計算式を記載した導入判断書を /company/cost-management.md に記録
□ 代替ツール 2 個以上の比較検討完了
□ データプライバシーポリシーを確認し、リスクがないことを確認
□ 月額コスト÷利用頻度の費用対効果が正 / 月 1000 円以下
□ 廃止時の移行コストが実装時間の 50% 以下であることを確認
```

### 4.2 ツール廃止の判断基準

**以下のいずれかに該当する場合、即座に廃止判定を実行**

| 条件 | 判定基準 | 実行タイミング |
|---|---|---|
| **コスト超過** | 月額費用 > 予算 | 毎月 1 日 |
| **利用頻度低下** | 月間利用日数 < 5 日 | 毎月末 |
| **代替ツール有無** | 無料で同等機能が存在 | 即座 |
| **ベンダー変更** | 利用規約 / 価格が不利 | 即座 |
| **ROI マイナス** | 効果測定で業績向上なし | 3 ヶ月ごと |

### 4.3 年次見直しチェックリスト

**毎年 1 月 1 日に実行**

```
□ 全ツール月額コストの合計確認
□ 廃止判定対象ツールの効果測定（3 ヶ月以上非利用ツール）
□ 新規無料ツール登場の有無調査（市場リサーチ）
□ セキュリティアップデート・利用規約変更の確認
□ 次年度の事業目標との整合性確認
□ 依存度「高」ツールの代替シナリオを更新
□ Repository の費用記録を次年度 `/company/cost-management.md` へ引継ぎ
```

## 5. データプライバシー原則

### 5.1 絶対禁止

❌ **以下のデータは AI / SaaS に入力しない**
- 実在する顧客名・企業名・社員氏名
- 金額・価格・見積書・提案書
- 会議記事・内部メモ・個人的な思考
- APIキー・トークン・認証情報

✅ **代わりに実施**
- 「某外資 SaaS、従業員 300 名規模の製造業」など匿名化
- 金額は「月額数万円」など概括化
- 機密情報は実装後に削除し、概要のみ記録

### 5.2 AI 入力時のチェックリスト

Claude Code へプロンプトを送信する前に確認

```
□ 顧客・企業・個人を特定する情報が含まれていないか
□ 金銭情報・価格・契約条件が含まれていないか
□ パスワード・APIキー・トークンが含まれていないか
□ 社内秘密・研究開発情報が含まれていないか
□ 個人の健康・家族情報が含まれていないか
```

### 5.3 ツール別データ取り扱いルール

| ツール | データ送信 | 保存 | 同期 |
|---|---|---|---|
| **Claude Code** | ✅ 匿名化済み情報のみ | ❌ Repository のみ | ❌ APIキー保存禁止 |
| **ChatGPT** | ✅ 匿名化済み情報のみ | ❌ コピペ手動運用 | ❌ トークン保存禁止 |
| **GitHub** | ✅ Markdown 形式（匿名化） | ✅ Public / Private | ⚠️ .gitignore で認証情報除外 |
| **Notion** | ✅ 非機密情報のみ | ✅ クラウド | ❌ API キー使用禁止 |
| **X / Note** | ✅ 公開予定のコンテンツのみ | ✅ パブリック | ✅ 必要に応じて手動投稿 |
| **Discord** | ✅ ホーム頻度高い話題のみ | ⚠️ クラウド | ❌ トークン / Webhook 保存禁止 |

### 5.4 定期監査

**毎月 15 日に実行**
- Repository に機密情報が誤ってコミットされていないか確認
- `.gitignore` が .env / .local.md / config 等を確実に除外しているか確認
- Notion への同期データが適切か確認
- X / Note 投稿にて個人情報が含まれていないか確認
