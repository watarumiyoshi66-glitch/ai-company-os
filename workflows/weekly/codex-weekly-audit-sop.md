# Codex Weekly Audit SOP

## 目的

Claude Code Remote と Codex の日次成果を週1回まとめ、翌週の発信・商品化・自動化改善に接続する。

## 実行タイミング

- 推奨：毎週日曜
- 対象期間：直近7日間
- 保存先：`/reports/YYYY-MM-DD-weekly-codex-audit.md`

## 確認対象

- `/research/market/`
- `/research/competitors/`
- `/intel/services-genai/`
- `/content/x/drafts/`
- `/content/note/research/`
- `/products/`
- `/reports/automation-memory/claude-codex-handoff-memory.md`
- `/tasks/owner-decisions.md`

## 手順

1. 今週の市場変化を3つに絞る
2. 伸ばすべきXテーマを3つ選ぶ
3. Note化すべきテーマを1〜3つ選ぶ
4. 商品化に近い仮説を1〜3つ選ぶ
5. 捨てるべきテーマ、保留すべきテーマを明記する
6. Claude Code Remoteの改善点をまとめる
7. オーナー判断事項を `tasks/owner-decisions.md` に追加する
8. 週次監査レポートを `/reports/YYYY-MM-DD-weekly-codex-audit.md` に保存する

## 判断基準

| 観点 | 良い状態 |
|---|---|
| 発信価値 | X投稿やNoteにしたとき、読者の認識が変わる |
| 収益接続 | 有料Note、テンプレート、ソフトウェアに接続できる |
| 差別化 | 競合と同じ話ではなく、外資IT営業の視点が入る |
| 根拠 | 出典URL、日付、未確認情報の区分がある |
| 継続性 | 来週も追う価値がある |

## 完了条件

- 週次監査レポートを保存済み
- 翌週のXテーマ、Noteテーマ、商品化仮説が明確
- Claude Code Remoteへの改善点を記録済み
- オーナー判断事項が整理済み
